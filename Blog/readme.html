A documentação está dividida em várias seções, abrangendo desde o início do projeto até como adicionar posts e personalizar o blog. --- # **Documentação do Projeto: Blog em Django** ## **1. Visão Geral** Este projeto é um simples **Blog** desenvolvido usando **Django**, uma poderosa framework Python. O blog permite a criação de posts, exibição de conteúdo, e é configurado com um painel de administração para facilitar a gestão do conteúdo. ## **2. Pré-requisitos** Antes de rodar o projeto, você precisa garantir que os seguintes requisitos estão instalados no seu sistema: - **Python** (versão 3.8 ou superior) - **Django** (versão 5.x) ### **Instalação do Django** Se você ainda não tem o Django instalado, basta executar o seguinte comando: ```bash pip install django ``` ## **3. Estrutura do Projeto** A estrutura do seu projeto deve ser parecida com esta: ``` Blog/ ├── blog/ │ ├── migrations/ │ ├── templates/ │ │ └── blog/ │ │ ├── lista_posts.html │ │ └── detalhe_post.html │ ├── __init__.py │ ├── admin.py │ ├── apps.py │ ├── models.py │ ├── views.py │ └── urls.py ├── Blog/ │ ├── __init__.py │ ├── asgi.py │ ├── settings.py │ ├── urls.py │ └── wsgi.py └── manage.py ``` ### **Principais Arquivos e Pastas** - **blog/**: Diretório do aplicativo principal (blog). - **Blog/settings.py**: Arquivo de configurações do Django. - **blog/templates/blog/**: Pasta onde ficam os templates do blog. - **blog/models.py**: Define os modelos de dados para o blog (como os posts). - **blog/views.py**: Contém as funções que gerenciam as requisições HTTP. - **blog/urls.py**: Define as URLs para as views do blog. ## **4. Configuração Inicial** ### **Passo 1: Criar o Projeto e o App** 1. **Criar o projeto Django**: No terminal, execute: ```bash django-admin startproject Blog cd Blog ``` 2. **Criar o aplicativo "blog"**: ```bash python manage.py startapp blog ``` ### **Passo 2: Configurar o App no Projeto** - Abra o arquivo `Blog/settings.py` e adicione o app `blog` na lista de `INSTALLED_APPS`: ```python INSTALLED_APPS = [ 'django.contrib.admin', 'django.contrib.auth', 'django.contrib.contenttypes', 'django.contrib.sessions', 'django.contrib.messages', 'django.contrib.staticfiles', 'blog', # Adicione esta linha ] ``` ### **Passo 3: Configurar a Base de Dados** 1. Certifique-se de que a configuração do banco de dados esteja configurada no arquivo `Blog/settings.py`. O Django, por padrão, utiliza o banco de dados SQLite, que é ideal para testes. 2. Execute as migrações para configurar as tabelas do banco de dados: ```bash python manage.py migrate ``` --- ## **5. Criando o Modelo de Post** No arquivo `blog/models.py`, definimos o modelo de dados para os posts do blog: ```python from django.db import models from django.utils import timezone class Post(models.Model): titulo = models.CharField(max_length=100) autor = models.CharField(max_length=100) conteudo = models.TextField() data_publicacao = models.DateTimeField(default=timezone.now) def __str__(self): return self.titulo ``` ### **Passo 1: Criar a Tabela no Banco de Dados** Execute o comando para criar as tabelas baseadas nos modelos: ```bash python manage.py makemigrations python manage.py migrate ``` --- ## **6. Criando o Painel de Administração** 1. No arquivo `blog/admin.py`, registre o modelo `Post` para que você possa gerenciar os posts diretamente no painel de administração do Django: ```python from django.contrib import admin from .models import Post admin.site.register(Post) ``` 2. Crie um superusuário para acessar o painel de administração: ```bash python manage.py createsuperuser ``` Após criar o superusuário, execute o servidor Django: ```bash python manage.py runserver ``` 3. Acesse o painel de administração em `http://127.0.0.1:8000/admin/` e faça login com o superusuário criado. Lá você poderá adicionar, editar e excluir posts. --- ## **7. Criando as Views e URLs** ### **Passo 1: Criar Views para Exibir os Posts** No arquivo `blog/views.py`, crie uma view para listar os posts e outra para mostrar o conteúdo de um post específico: ```python from django.shortcuts import render from .models import Post def lista_posts(request): posts = Post.objects.all() return render(request, 'blog/lista_posts.html', {'posts': posts}) def detalhe_post(request, post_id): post = Post.objects.get(id=post_id) return render(request, 'blog/detalhe_post.html', {'post': post}) ``` ### **Passo 2: Configurar as URLs** No arquivo `blog/urls.py`, defina as URLs para essas views: ```python from django.urls import path from . import views urlpatterns = [ path('', views.lista_posts, name='lista_posts'), path('post//', views.detalhe_post, name='detalhe_post'), ] ``` No arquivo `Blog/urls.py`, inclua as URLs do app `blog`: ```python from django.contrib import admin from django.urls import path, include urlpatterns = [ path('admin/', admin.site.urls), path('', include('blog.urls')), ] ``` --- ## **8. Criando os Templates** No diretório `blog/templates/blog/`, crie os templates para exibir a lista de posts e os detalhes de um post. ### **Template para Listar os Posts (`lista_posts.html`)** ```html {% load static %} Blog

Lista de Posts

    {% for post in posts %}
  • {{ post.titulo }}
    • {% endfor %}
``` ### **Template para Detalhes do Post (`detalhe_post.html`)** ```html {% load static %} {{ post.titulo }}

{{ post.titulo }}

Autor: {{ post.autor }}

Data: {{ post.data_publicacao }}

{{ post.conteudo }}

Voltar para a lista de posts ``` --- ## **9. Estilizando o Blog** Crie um arquivo CSS no diretório `blog/static/app/content/styles.css` para estilizar seu blog de forma moderna. Exemplo básico de estilo: ```css body { font-family: Arial, sans-serif; background-color: #f4f4f4; color: #333; margin: 0; padding: 0; } h1 { text-align: center; color: #5a5a5a; } ul { list-style-type: none; padding: 0; } li { background-color: #fff; margin: 10px; padding: 15px; border-radius: 5px; } a { text-decoration: none; color: #007bff; } a:hover { text-decoration: underline; } ``` --- ## **10. Conclusão** Agora o seu blog está configurado e funcionando.
